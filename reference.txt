-자전거 반납 및 식당 예약 서비스 연계 기능

자전거 반납 후에는 ~ -> 새 페이지로 이동
사용자 위치 정보를 기반으로 ~ -> 외부 GPS actor 을 통해 사용자의 위치 정보를 얻음 -> 시스템 내부 기능 -> list, description에 작성 X
근처 식당을 추천 받아서 ~ -> DB에 지역별 맛집 저장 -> 시스템 내부 기능 / 
예약할 수 있는 ~ -> "근처 맛집 예약하기!" 버튼으로 맛집 리스트 출력 페이지로 이동 -> 각 맛집에 예약하기 버튼 존재(extend)
외부 서비스와 연결된다 ~ -> 각 맛집별 예약하기 버튼과 외부 예약 서비스 actor 연결 -> 시스템 내부 기능
대기 예약한 회원이 있는 경우 ~ -> 예약 대기 기능에서 버튼을 누르면 DB에 예약 순서 저장 -> 시스템 내부 기능
-> 예약 대기 회원에게 이메일 보내는 건, 반납 관련된 외부 actor
-> 내부 시스템에서,
ex) 자전거 1 반납 -> 해당 자전거 예약리스트 조회 -> 해당 회원 정보를 이메일 시스템에 파라미터로 넘김 -> 이 과정 시스템 내부 기능
외부 actor 3개 모두 내부 기능이 requirement, description 작성 필요 X, Use case에서 외부 actor와 연결만 표시
